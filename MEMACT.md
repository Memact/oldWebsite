# Memact Website Notes

Memact is where users see what apps know about them and control it.

Current public line:

```text
Your Identity. Your Choice.
See what apps know about you and control it.
```

Website is the portal for users and developers. It handles sign in, developer
app setup, API keys, consent pages, Yourself, account settings, help, SEO, and
public learn content.

## System Position

```text
Website manages -> Access checks -> Context organizes -> Yourself shows -> Memory stores -> SDK returns allowed memory -> Apps personalize
```

Website is not the Context engine or Memory store. It is the control surface.

## Copy Rules

Use simple public copy:

- "See what apps know about you and control it."
- "You choose what an app can use."
- "Yourself shows what apps suggest, what you accept, and what you can change."
- "Apps should not start from zero, and they should not invent identity from weak activity."

Keep technical terms in developer sections:

- memory suggestions
- Context category rules
- SDK
- scopes and categories
- source trail

Avoid:

- retired tagline wording
- intent-first positioning
- Playground as a current repo/product area
- Capture or Inference as current core product language
- AI platform hype
- infra soup in public copy
- claims that apps receive private data by default
